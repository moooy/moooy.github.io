webpack相关使用模块

  • path模块
    NodeJS使用path模块来处理文件路径的问题。

    • 规范化连接：path.normalize()
    • 解析路径: path.resolve(),将多个路径解析为一个规范化的绝对路径
    • 绝对路径到相对路径的转化
    • 提取路径的组成部分
    • 判断路径是否存在
      __dirname始终指向当前js代码文件的目录。

  • 实时刷新
    问题提出：每次更改main.js文件或其引用的其他模块，都需要手动对bundle.js使用webpack命令或npm run build来重新生成bundle.js文件

    • 在项目根目录下执行webpack –watch来监控目录下的文件变化并进行实时的重新构建。
    • webpack 提供了 webpack-dev-server 解决实时刷新页面的问题，同时解决实时构建的问题。
